# 系统架构文档

## 概述

OpenClaw Telegram 私聊总结系统是一个基于 OpenClaw 会话数据的自动化总结生成系统。系统通过解析 Telegram 私聊对话，使用 AI 模型生成结构化总结，并通过定时任务自动执行。

## 核心组件

### 1. 数据收集层

#### JSONL 解析器 (`jsonl_parser.py`)
- **功能**: 解析 OpenClaw 生成的 JSONL 会话文件
- **输入**: `.jsonl` 文件路径
- **输出**: `Session` 对象列表
- **关键方法**:
  - `parse_session_file()`: 解析单个会话文件
  - `parse_all_sessions()`: 解析所有会话文件
  - `list_session_files()`: 列出所有会话文件

#### 数据模型 (`data_models.py`)
- **Message**: 消息数据模型
  - `message_id`: 消息ID
  - `user_id`: 用户ID
  - `text`: 消息内容
  - `timestamp`: 时间戳
  - `message_type`: 消息类型
- **Session**: 会话数据模型
  - `session_id`: 会话ID
  - `display_name`: 显示名称
  - `messages`: 消息列表
  - `participants`: 参与者列表

### 2. 数据处理层

#### 数据管理器 (`data_manager.py`)
- **功能**: 管理数据的收集、过滤和归档
- **核心方法**:
  - `collect_daily_data()`: 收集每日数据
  - `collect_weekly_data()`: 收集每周数据
  - `archive_raw_data()`: 归档原始数据
  - `load_user_summaries()`: 加载用户总结

#### 数据过滤逻辑
```python
# 按日期过滤消息
daily_messages = [
    msg for msg in session.messages
    if msg.timestamp.date() == target_date.date()
]
```

### 3. AI 总结层

#### AI 总结器 (`ai_summarizer.py`)
- **功能**: 使用 AI 模型生成对话总结
- **配置**:
  - 模型: DeepSeek Chat
  - 最大 tokens: 2000
  - 温度: 0.7
- **总结类型**:
  - 每日总结: 重点突出当天关键话题和决策
  - 每周报告: 综合分析本周趋势和进展

#### 提示词模板
```python
DAILY_SUMMARY_PROMPT = """
基于以下对话记录，生成每日总结：
{conversation}

请包括：
1. 关键话题讨论
2. 提到的任务
3. 做出的决策
4. 下一步行动计划
"""
```

### 4. 任务调度层

#### 主程序 (`main.py`)
- **入口点**: `TelegramSummarySystem` 类
- **运行模式**:
  - `--daily`: 运行每日总结
  - `--weekly`: 运行每周报告
  - `--status`: 显示系统状态

#### 调度器 (`scheduler.py`)
- **功能**: 管理定时任务执行
- **集成**: 与 OpenClaw cron 系统集成
- **任务类型**:
  - 一次性任务
  - 周期性任务
  - 延迟任务

### 5. 配置管理

#### 配置文件 (`config/config.yaml`)
```yaml
# 数据源配置
data_source:
  sessions_path: "/root/.openclaw/agents/main/sessions"
  source_type: "direct"  # direct/group

# AI 配置
ai:
  model: "deepseek/deepseek-chat"
  max_tokens: 2000

# 存储配置
storage:
  base_path: "/path/to/data"
  raw_data: "raw"
  processed_data: "processed"
```

### 6. 存储层

#### 目录结构
```
data/
├── raw/                    # 原始数据
│   └── daily/             # 每日原始数据
│       └── YYYY-MM-DD/    # 按日期组织
├── processed/             # 处理后的数据
│   └── by_user/          # 按用户组织
│       └── <user_id>/    # 用户目录
│           └── daily_summary_YYYY-MM-DD.json
└── summaries/            # 总结文件
    └── daily/            # 每日总结
```

### 7. 通知层

#### Telegram 通知
- **推送方式**: 通过 OpenClaw Telegram 插件
- **消息格式**: Markdown 格式的总结报告
- **触发条件**: 任务完成时自动发送

## 数据流

### 每日总结流程
1. **触发**: 定时任务触发 (08:00 UTC)
2. **收集**: 收集前一天的对话数据
3. **过滤**: 按日期过滤相关消息
4. **归档**: 保存原始数据到文件系统
5. **处理**: AI 生成总结
6. **存储**: 保存总结到 JSON 文件
7. **通知**: 发送 Telegram 通知

### 每周报告流程
1. **触发**: 定时任务触发 (周五 18:00 UTC)
2. **收集**: 收集本周的对话数据
3. **分析**: 综合分析本周趋势
4. **生成**: 生成周度报告
5. **存储**: 保存报告文件
6. **通知**: 发送周报通知
